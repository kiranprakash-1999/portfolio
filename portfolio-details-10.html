Cloud Lighting System/Weather Station

  • Category: Project
  • Members: Kiran Prakash, Sophie Kenny
  • Project Timeline: 2 Weeks; December, 2020

An indoor cloud lighting system that is responsive to the weather via sensor data

For a school project, our team built a lighting system that reacts to sensor data and lights a WS2811 LED strip. The strip is constructed within an lantern, designed to look like a cloud. This sytem acts as a weather station and a lighting system with more diffused, natural lighting which indicate the weather outside. The system includes analog sensors with signals divided and inputted in the MSP430 MCU. The MCU interfaces between the sensor circuit, the LED strip, and a simple windows form GUI. A user can manually set a colour or can have the lights react to the weather. Firmware was built to utilized the MCUs ADCs, timers, UART and SPI modules. UART used for communicating with the GUI, and SPI for the WS2811 ICs. Timers were used for the main operations of the MCU. This involved 1) sending, receiving, and validating packets 2) acquiring values from the ADC and using it to check the type of weather 3) sending indiviual bits to WS2811 via SPI to configure and activate the LED strip This project taught me much about systematically debugging machines. Isolating mechanical, electrical, and programmed elements to try and replicate and isolate the bug. Reducing the firmware back to a working stage, and slowly and purposefully readding code and ensuring it works as intended. This project also developed my habit to read online documentation and datasheets more rigorously.
